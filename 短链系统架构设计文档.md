## 1. 项目概述

短链服务（URL Shortener）是将长链接转换为短链接的在线服务，核心本质是建立并维护 **短码 ↔ 长链接** 的映射关系。用户访问短链时，系统通过短码快速查找原始长链接并返回 302 重定向。

### 1.1 核心价值

| 价值点  | 说明                              |
| ---- | ------------------------------- |
| 链接精简 | 显著缩短 URL 长度，便于在短信、社交媒体等字符受限场景分享 |
| 用户体验 | 短链更美观，提升点击意愿                    |
| 数据追踪 | 支持点击量统计、来源分析                    |
| 链接管理 | 支持过期时间设置、状态禁用/启用                |
| 品牌赋能 | 可绑定自定义域名，增强品牌信任感                |
| 部署简单 | 单一可执行文件，零配置部署                   |
| 极速跳转 | 纯 SQLite 本地查询，毫秒级响应，无网络延迟       |

### 1.2 核心业务流程

**生成短链（写）：**

```
长链接 → 合法性校验 → 生成唯一ID → Base62编码 → 存储映射 → 返回短链
```

**访问短链（读）：**

```
短链 → 提取短码 → 直接查询SQLite → 302重定向 → 异步记录访问统计
```

```sql
CREATE TABLE short_urls (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    short_code VARCHAR(12) UNIQUE NOT NULL,
    name VARCHAR(100),
    remark VARCHAR(500),
    long_url VARCHAR(2048) NOT NULL,
    user_id INTEGER DEFAULT 0,
    visit_count INTEGER DEFAULT 0,
    expire_at DATETIME,
    status INTEGER DEFAULT 1,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_short_code ON short_urls(short_code);
CREATE INDEX idx_user_id ON short_urls(user_id);
CREATE INDEX idx_expire_at ON short_urls(expire_at);
CREATE INDEX idx_status ON short_urls(status);
```

**设计要点：**

- `short_code` 添加 UNIQUE 唯一索引，防止重复
- `name` 短链名称，方便识别用途
- `remark` 备注字段，记录更多信息
- `long_url` 长度设为 2048，兼容绝大多数 URL 标准
- `status`: 1-正常，0-禁用，2-已过期

## 4. 核心模块详细设计

### 4.1 短码生成算法

```go
import (
    "github.com/bwmarrin/snowflake"
)

const base62Chars = "0123456789abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ"

var node *snowflake.Node

func init() {
    node, _ = snowflake.NewNode(1)
}

func Base62Encode(num int64) string {
    if num == 0 { return "0" }
    var result []byte
    base := int64(62)
    for num > 0 {
        remainder := num % base
        result = append([]byte{base62Chars[remainder]}, result...)
        num = num / base
    }
    return string(result)
}

func GenerateShortCode() string {
    id := node.Generate().Int64()
    obfuscated := id ^ 0xAAAAAAAAAAAAAAA
    return Base62Encode(obfuscated)
}
```

### 4.2 短链查询 —— 极速直连数据库

```go
func (s *ShortURLService) GetLongURL(shortCode string) (string, error) {
    // 参数校验
    if !util.IsValidShortCode(shortCode) {
        return "", ErrShortCodeNotFound
    }

    // 直接查询 SQLite 数据库（毫秒级响应）
    shortURL, err := s.repo.FindByShortCode(shortCode)
    if err != nil {
        if errors.Is(err, gorm.ErrRecordNotFound) {
            return "", ErrShortCodeNotFound
        }
        return "", err
    }

    // 检查状态
    if shortURL.Status != model.StatusActive {
        return "", ErrShortCodeNotFound
    }

    // 检查过期
    if shortURL.ExpireAt != nil && shortURL.ExpireAt.Before(time.Now()) {
        shortURL.Status = model.StatusExpired
        s.repo.Update(shortURL)
        return "", ErrShortCodeExpired
    }

    // 异步更新计数（本地内存聚合）
    s.counter.Add(shortCode)
    return shortURL.LongURL, nil
}
```

**性能优势：**
- 无外部缓存依赖，减少网络IO
- SQLite 查询毫秒级响应
- 本地内存计数聚合，减少数据库写入压力

### 4.3 访问计数 —— 本地聚合批量刷盘

```go
// 本地内存计数聚合器
type CounterAggregator struct {
    mu     sync.Mutex
    data   map[string]int64
    ticker *time.Ticker
    repo   repository.ShortURLRepository
}

func NewCounterAggregator(repo repository.ShortURLRepository) *CounterAggregator {
    c := &CounterAggregator{
        data:   make(map[string]int64),
        ticker: time.NewTicker(5 * time.Second),
        repo:   repo,
    }
    go c.run()
    return c
}

func (c *CounterAggregator) Add(shortCode string) {
    c.mu.Lock()
    defer c.mu.Unlock()
    c.data[shortCode]++
}

func (c *CounterAggregator) run() {
    for range c.ticker.C {
        c.Flush()
    }
}

func (c *CounterAggregator) Flush() {
    c.mu.Lock()
    batch := c.data
    c.data = make(map[string]int64)
    c.mu.Unlock()
    
    // 批量更新数据库
    for code, count := range batch {
        c.repo.IncrementVisitCount(code, count)
    }
}
```

**设计优势：**
- 纯本地内存实现，无外部依赖
- 批量刷盘减少数据库写入压力
- 5秒定时刷新，保证数据最终一致性

## 7. Docker 部署

### 7.1 Dockerfile

```dockerfile
# 构建阶段：使用 Alpine 基础镜像，纯 Go 编译无需 CGO
FROM golang:1.25-alpine AS builder

WORKDIR /app

# 安装依赖
COPY go.mod go.sum ./
RUN go mod download

# 编译：CGO_ENABLED=0，纯 Go 编译，无外部依赖
COPY . .
RUN CGO_ENABLED=0 GOOS=linux go build -o minlink ./cmd/main.go

# 运行阶段：使用 Alpine 基础镜像，体积约 20MB
FROM alpine:latest

WORKDIR /app

# 复制编译产物和静态资源
COPY --from=builder /app/minlink .
COPY --from=builder /app/static ./static

# 创建数据目录
RUN mkdir -p /app/data

# 设置时区
RUN apk --no-cache add tzdata && cp /usr/share/zoneinfo/Asia/Shanghai /etc/localtime

EXPOSE 8080

CMD ["./minlink"]
```

**Docker 优化说明：**

| 优化项 | 原方案 | 新方案 | 效果 |
| ---- | ---- | ---- | ---- |
| **SQLite 驱动** | mattn/go-sqlite3 (CGO) | glebarez/sqlite (纯 Go) | 消除 CGO 依赖 |
| **编译模式** | CGO_ENABLED=1 | CGO_ENABLED=0 | 纯静态编译 |
| **基础镜像** | Debian (≈120MB) | Alpine (≈20MB) | 镜像体积减少 80%+ |
| **运行时依赖** | 需要 sqlite-libs | 无需任何运行时依赖 | 零依赖运行 |

## 8. 性能优化策略

### 8.1 架构优化

| 优化点           | 方案                          | 效果              |
| -------------- | --------------------------- | --------------- |
| **去除外部缓存依赖**   | 直接查询 SQLite               | 减少网络IO，毫秒级响应   |
| **本地计数聚合**     | 内存批量聚合 + 定时刷盘             | 减少数据库写入压力      |
| **Snowflake ID** | 本地生成唯一ID                   | 无网络依赖，高性能      |
| **索引优化**       | short_code、user_id 等关键字段索引 | 查询性能提升         |
| **纯 Go SQLite 驱动** | glebarez/sqlite (无 CGO)         | 编译简单，Alpine 友好，镜像体积减少 80%+ |

### 8.2 查询性能

- SQLite 查询响应时间：**< 1ms**
- 短链跳转总耗时：**< 10ms**
- 无外部服务调用延迟

### 8.3 写入优化

- 访问计数本地聚合，5秒批量刷盘
- 减少数据库写入频率，降低IO压力
